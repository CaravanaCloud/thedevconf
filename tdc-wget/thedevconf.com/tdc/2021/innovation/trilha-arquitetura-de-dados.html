Introduction to data streaming

Nicolas Fränkel Frankel

The batch processing model has been serving us for decades. Yet, it might have reached its end-of-life for most use-cases. As the pace of businesses increases, decision makers prefer slightly wrong data sooner, than 100% accurate data later. Stream processing matches this usage: instead of managing the entire bulk of data, manage pieces of them as soon as they become available.

In this talk, I?ll define the context of the batch processing model, the reasons behind stream processing, how they compare, what are their pros and cons, and a list of existing technologies implementing the latter. I?ll conclude by a demo that displays in (near) real-time all trains on a map using OpenData.

William Lino Oliveira

Será uma abordagem mão na massa utilizando bancos de dados em Kubernetes sendo armazenados em volumes com StorageOS.

Organize seus dados com grafos de conhecimento

Rafael Rocha

Grafos de conhecimento é uma forma de armazenar e organizar dados mantendo semântica e contexto das informações. Os repositórios de grafos de conhecimento são livres de esquema permitindo o crescimento dos dados. O contexto é mantido pela nomeação dos grafos (named graph). Ontologia conserva a semântica dos grafos. Esses padrões tornam os grafos de conhecimento ideal como base para análise de dados e analytics.

Aplicação de um modelo de dados orientado a eventos

Douglas José Araujo / Adriano Ribeiro

A forma como modelamos o estado da aplicação restringe ou amplia, simplifica ou burocratiza o processo de uso dos dados para atender aos requisitos de negócio. Os problemas que nossos clientes enfrentam são muito complexos para simplesmente traduzirmos em um grupo pequeno de operações atômicas com insert, update, select ou delete considerando um tipo de operações padronizadas. Esse ?estado? é o conjunto de objetos que criamos e que representam as características duráveis do nosso negócio, e que precisamos armazenar em algum lugar para recuperar posteriormente, e quando utilizamos um banco de dadtos para isso, não importa se ele é relacional e nosql, este estado tende a ser transiente.
